# Contributing to Bongo AI

Thank you for your interest in contributing to Bongo AI! This document provides guidelines and instructions for contributing.

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers and encourage diverse perspectives
- Focus on constructive feedback
- Prioritize the learning experience of Bangladesh students

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/Bongo-AI.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes
6. Commit: `git commit -m "Description of changes"`
7. Push: `git push origin feature/your-feature-name`
8. Create a Pull Request

## Development Guidelines

### Code Style

- Follow existing code patterns
- Use TypeScript for type safety
- Write meaningful variable and function names
- Add comments for complex logic
- Keep functions small and focused

### Commit Messages

- Use present tense ("Add feature" not "Added feature")
- Be descriptive but concise
- Reference issues when applicable

### Testing

- Write tests for new features
- Ensure all tests pass before submitting PR
- Maintain or improve code coverage

### Bengali Content

- All user-facing strings must support Bengali (bn-BD)
- Test Bengali text rendering in UI
- Ensure proper RTL/LTR handling
- Use Bengali examples in documentation where appropriate

## Pull Request Process

1. Update README.md with details of changes if needed
2. Update documentation for new features
3. Ensure all tests pass
4. Request review from maintainers
5. Address review feedback
6. Wait for approval and merge

## Areas for Contribution

- Bengali content and translations
- Educational content for Bangladesh curriculum
- AI model improvements for Bengali language
- UI/UX improvements
- Bug fixes
- Documentation
- Testing
- Performance optimization

## Questions?

Feel free to open an issue for questions or discussions.
